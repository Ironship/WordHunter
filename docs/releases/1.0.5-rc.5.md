# Word Hunter 1.0.5-rc.5

> **Warning:** This is a prerelease intended only for testing. It may contain
> bugs. Back up your Word Hunter data before installing and report regressions
> through GitHub Issues.

## Changes Since RC4

- The Pocket selected-word sheet can now slide much farther down, leaving only
  a compact card preview above the bottom navigation.
- Releasing the handle at an intermediate height preserves that exact relative
  position instead of forcing the sheet to one of two endpoints. The position
  remains stable while swiping through words and is recalculated after viewport
  changes such as device rotation.
- The complete selected-word card now uses a theme-aware tint matching its New,
  Learning, Known, or Ignored status, resolving GitHub issue #60. Status labels,
  buttons, accessibility state, and outgoing-card colors update together.
- Loading and empty Reader states no longer retain the previous word's color.
- Publishing a validated draft no longer starts a duplicate full artifact build.
  Manual release builds are required to match the draft's target revision.

## Included From Earlier RCs

- Complete TypeScript browser runtime and deterministic generated frontend.
- Configurable selected-word panel visibility and ordering.
- Complete word selection and automatic translation after Reader swipes.
- Layered Reader and Flashcard card gestures with reduced-motion support.
- Desktop theme and Edit Book layout corrections plus current-week Pocket
  heatmaps.

## Update Policy

GitHub marks this build as a prerelease. Stable Word Hunter installations query
only GitHub's latest stable release endpoint, so this test build does not trigger
the in-app new-version popup.
