# Word Hunter 1.0.5-rc.6

> **Warning:** This is a prerelease intended only for testing. It may contain
> bugs. Back up your Word Hunter data before installing and report regressions
> through GitHub Issues.

## Changes Since RC5

- Word statuses changed while synchronization is running are no longer replaced
  by an older backend snapshot. Snapshot reads remain unacknowledged until the
  frontend accepts the exact local revision, and queued saves merge against the
  last state actually seen by the frontend.
- Background synchronization saves pending fields first, rejects stale Desktop
  and Android snapshots, and retries without a tight loop.
- `Ctrl+Enter` now returns to the exact selected Reader word, including from
  translation and note fields, and never falls back to the first word on the
  page. Dialogs and repeated keydown events remain protected.
- Delayed native SRS grades no longer overwrite a newer word status or apply to
  the same spelling in a different language profile.
- These changes resolve duplicate GitHub issues #61 and #62.

## Included From Earlier RCs

- Extended Pocket word-sheet drag range with persistent intermediate heights.
- Theme-aware complete-card tints for New, Learning, Known, and Ignored words.
- Complete TypeScript browser runtime and deterministic generated frontend.
- Configurable selected-word panel visibility and ordering.
- Layered Reader and Flashcard card gestures with reduced-motion support.
- Desktop theme and Edit Book corrections plus current-week Pocket heatmaps.

## Update Policy

GitHub marks this build as a prerelease. Stable Word Hunter installations query
only GitHub's latest stable release endpoint, so this test build does not trigger
the in-app new-version popup.
