# Word Hunter 1.0.5-rc.7

> **Warning:** This is a prerelease intended only for testing. It may contain
> bugs. Back up your Word Hunter data before installing and report regressions
> through GitHub Issues.

## Changes Since RC6

- The Pocket selected-word sheet can now be dragged down across roughly four
  fifths of the screen, retains intermediate positions, and no longer jumps or
  changes its lower limit when Smart Suggestion appears or disappears.
- Horizontal word-card swipes require less effort and keep vertical scrolling
  available. Pocket navigation uses larger targets positioned lower on tall
  phones for easier one-handed reach.
- Every word now has a visible optional Article field in the Reader and Add/Edit
  dialog. Articles are persisted per language profile and displayed with the
  headword in Vocabulary, text-to-speech, and flashcards.
- Smart Suggestion can save grammatical articles separately from the vocabulary
  key for German, French, Spanish, and Italian. French and Italian forms such as
  `l'homme`, `l’homme`, and `un’amica` resolve to the bare word while
  older attached-article keys remain readable.
- Vocabulary search, text filtering, TXT export, and Anki TSV import/export now
  include the optional article. The fourth TSV column remains backward
  compatible with existing three-column files.
- Reverse flashcards mask both the word and its article, including typographic
  apostrophes and sentences where an adjective separates the article and noun.

## Included From Earlier RCs

- Synchronization revision guards and exact Ctrl+Enter Reader restoration.
- Extended Pocket word-sheet gestures and complete-card status tints.
- Complete TypeScript browser runtime and deterministic generated frontend.
- Configurable selected-word panel visibility and ordering.
- Desktop theme, Edit Book, heatmap, and SRS consistency fixes.

## Update Policy

GitHub marks this build as a prerelease. Stable Word Hunter installations query
only GitHub's latest stable release endpoint, so this test build does not trigger
the in-app new-version popup.
