# Word Hunter 1.0.5-rc.4

> **Warning:** This is a prerelease intended only for testing. It may contain
> bugs. Back up your Word Hunter data before installing and report regressions
> through GitHub Issues.

## Changes Since RC3

- Reader word swipes now use the complete selection path, including exact token
  context, automatic translation when enabled, and a single spoken-word action.
  Keyboard and toolbar navigation remain lightweight and do not add every
  traversed word to the vocabulary.
- The Pocket selected-word panel is now a draggable bottom sheet with collapsed
  and expanded snap points, velocity-aware release behavior, and an accessible
  keyboard-operable handle.
- Horizontal card gestures now track the finger, preserve the outgoing card as
  a non-interactive visual layer, and show the newly selected word immediately.
- Translation responses no longer refresh a different selected word, and
  duplicate in-flight automatic translations are suppressed.
- Sheet and card transitions respect reduced-motion preferences, preserve pinch
  zoom, and remain isolated from desktop Reader layout.

## Included From Earlier RCs

- Complete TypeScript browser runtime and deterministic generated frontend.
- Configurable selected-word panel visibility and ordering.
- Desktop theme and Edit Book layout corrections.
- Directional Reader gestures, card-first Flashcards, and current-week Pocket
  heatmaps.

## Update Policy

GitHub marks this build as a prerelease. Stable Word Hunter installations query
only GitHub's latest stable release endpoint, so this test build does not trigger
the in-app new-version popup.
